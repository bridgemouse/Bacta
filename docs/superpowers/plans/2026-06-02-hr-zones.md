# HR Zones Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Capture Garmin heart rate zone minutes in the poller/ingest, expose via API, and display a proportional zone bar in the Training Overview.

**Architecture:** Four layers in dependency order — server allowlist first, then TypeScript interface, then hook, then UI. Python poller/ingest changes last (data capture). A backfill run populates historical data so the UI panel appears immediately.

**Tech Stack:** Python (garmin-connect), Express/TypeScript, React 19, Vitest + Testing Library, better-sqlite3.
